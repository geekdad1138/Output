Wow. So GitHub just dropped a new app, GitHub Copilot. Not the CLI, not vscode with the integration... just GitHub Copilot the app.

It's in technical preview so not generally available yet but I got a chance to play with it.

Buttons to run your app, optionally open the files it's changing in an IDE, integrations for PRs, branches, repos. Customize skills, hook up to MCP servers, set up automated workflows... lots of fun toys to start playing with.

I gave it a test, told it to make an update on a website using natural language, almost like you'd see in a support ticket. Not like a coder who knew the tech stack for the site. "This website is slow because every time you check a box it refreshes the page." It wrote up some very clean, well formatted client side functions, WITH COMMENTS. I tested it, and it worked.

I don't know if we're quite to the place yet where we can completely walk away from the code, but if people truly adopt this tool, it forces you to abstract your thinking away from "On line 36 - refactor this function" and more to "this app/tool/feature needs this addition/change/fix" and then TRUST that the AI model you chose can understand the code base to make the change for you.
